## Summary
Modified `TemplateServiceImpl.printQuote()` to return a base64-encoded PDF in both success and fallback paths. When no matching document template is found, a `mock.pdf` is loaded from classpath and returned as base64, allowing the frontend to handle both success and fallback uniformly without a separate GET endpoint.

## Affected APIs
- `POST /api/template/printQuote` â€” now always returns JSON with `createdPdf.binary` (base64)

## Implementation Details

### Backend Changes

- Success path: wraps MMP binary response bytes as `createdPdf.binary` base64 string

- Added `loadMockPdfBase64()` private method that reads `mock.pdf` from classpath resources and encodes as Base64
