## Summary
Fixed inconsistency between UI text and Hyperfront terminology by changing the word "quotation" to "proposal" in the "Modify Deal" popup confirmation message.

## Problem
The word "quotation" was displayed in the "Modify Deal" popup confirmation message, but Hyperfront terminology uses "proposal" instead of "quotation". This caused inconsistency between the UI text and the established Hyperfront terminology.

## Solution
Changed the `confirmationModifyDealMessage` i18n string from "quotation" to "proposal" in both locale files:
- `locales/en-US.json` - line 990
- `locales/fr-FR.json` - line 990

## Changes Made

### Before
```json
"confirmationModifyDealMessage": "A new version of your quotation will be created. The previous version will remain visible with a 'Cancelled' status."
```

### After
```json
"confirmationModifyDealMessage": "A new version of your proposal will be created. The previous version will remain visible with a 'Cancelled' status."
```

## Modified Files
- `locales/en-US.json` (line 990)
- `locales/fr-FR.json` (line 990)
