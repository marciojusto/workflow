## Bug Fix ‚Äî VAT percentage not persisted to MongoDB

**Discovered:** 2026-04-17
**Fixed:** 2026-04-17

### Root Cause

When a deal is created, `buildDealPayload()` saved `quote_calculation` with only `{ options: [...] }` ‚Äî no `vatPercentage`, no `vatPercent`. The `mappingCreateDealResponse()` correctly computed `vatPercentage` and returned it to the frontend API response, but this value was **never written to the MongoDB `quote_calculation` document**.

Additionally, `dealService.ts` `updateDealCalculation()` saved `vat_percent` (enumId like "VAT_21") as a top-level deal field but **NOT** `vat_percentage` (the actual number like 21). When `populateAssetFromDeal()` loaded the deal from MongoDB, `quoteCalc.vatPercentage` was always `null`, and there was no fallback to a top-level field.

**Symptom:** VAT percentage showed "-" for ALL deals in the Summary section, even newly created ones.

After a calculate call, `vatPercentage` was included in `quote_calculation` (because `buildCalculateResponse` returns it in `finalResponse.data`), but only if the deal had been calculated at least once. Newly created deals that were never calculated would always show "-".

### Fix Applied

1. **`server/services/dealPayloadBuilder.ts`** ‚Äî Made `buildDealPayload()` async, imported `getVatPercentageFromMongo()`, extracted `vatEnumId` from `carConfig`, computed `vatPercentage` at deal creation time. Added `vatPercentage` + `vatPercent` to `quote_calculation` object, and added `vat_percentage` as a top-level field on the deal document.

2. **`server/utils/mappingResponse.ts`** ‚Äî Exported `getVatPercentageFromMongo()` so it can be imported by `dealPayloadBuilder.ts`.

3. **`server/services/dealService.ts`** ‚Äî Added `vat_percentage: calculationData?.vatPercentage` to the `$set` in `updateDealCalculation()`, ensuring it's persisted on calculate too.

4. **`composables/useInit.ts`** ‚Äî Added fallback chain `quoteCalc?.vatPercentage ?? dealData.vat_percentage ?? null` in `populateAssetFromDeal()`, so if `quote_calculation.vatPercentage` is missing, the top-level `vat_percentage` field is used.

5. **`server/api/deals/index.post.tsx`** ‚Äî Changed `buildDealPayload()` call to `await` since the function is now async.

6. **`server/api/deals/duplications.post.tsx`** ‚Äî Changed `buildDealPayload()` call to `await` and converted positional args to named object args (matching the `BuildDealPayloadArgs` type).

### Existing Deals

Deals created before this fix (e.g., 10003721) will not have `vat_percentage` in their MongoDB document. They will get it after the next calculate call, or a MongoDB migration script could be run to backfill.
