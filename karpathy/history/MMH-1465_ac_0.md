## Implementation Summary

Introduced a per-party `lockedPartyIndices` state in the offer store to track which parties have been saved and should have their fields locked in the Proposal phase.

- `features/offers/stores/offer.ts`: Added `lockedPartyIndices: number[]`, `lockParty(index)`, and `unlockParty(index)` actions.
- `features/party/Third.vue`: Added `isPartyLocked` computed property and integrated it into `updatePartyReadonlyFields()` so that when a party is locked in Proposal phase, all fields become read-only.
- `features/deal/Offer.vue`: After a successful global save, all dirty party indices are locked.
